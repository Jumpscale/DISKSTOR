/nos/reserve post

post

Make a reservation for specific amount of storage in the NOS

post /nos/reserve

Make a reservation for specific amount of storage in the NOS

  • Request
  • Response

Body

Media type: application/json

Type: object

Properties
  • IYO_UID: required (string)

    Identity of reserver. Will have full rights for reservation.

  • reservationId: (string)

    Unique id (upto 16 chars) which identifies the reservation. If not passed will be autogenerated by server.

  • reservationAdminSecret: (string)

    Secret to control the reservation. If not set - reservation is publicly available.

  • size: required (integer)

    Size of reservation in megabytes.

  • expirationEpoch: (integer)

    All data removed after expiration date. Epoh format.

Examples:

Regular reservation, reservationId generated by server and returned:

{
  "IYO_UID": "6a6f686e646f656a756e696f7274776f",
  "size": 100
}

Private reservation with expiration:

{
  "IYO_UID": "6a6f686e646f656a756e696f7274776f",
  "reservationId": "D9kdMquM0ElYvnQ3",
  "reservationAdminSecret": "MqydbtJK/kdlJJ7RjW+5aS25ZlVKWvcWoLY2QXbEr6o=",
  "size": 50,
  "expirationEpoch": 1521532065
}

Public reservation:

{
  "IYO_UID": "6a6f686e646f656a756e696f7274776f",
  "reservationId": "MyPublicRes1",
  "size": 300
}

HTTP status code 200

Reservation successfull

Body

Media type: application/json

Type: object

Properties
  • reservationId: required (string)

    reservationId passed in parameters or generated by the server if absent in parameters

Example:

{
  "reservationId": "D9kdMquM0ElYvnQ3"
}

HTTP status code 409

ReservationId is not unique.

HTTP status code 413

Not enough space available.

/nos/put post

post

Put data into storage

post /nos/put

Put data into storage

  • Request
  • Response

Body

Media type: application/json

Type: object

Properties
  • IYO_UID: required (string)

    Identity of requeter.

  • reservationId: required (string)

    Unique ID of the reservation.

  • dataSecret: (string)

    Secret with W access level to the specific reservation. Pass empty string or ignore if reservation is public.

  • key: required (string)

    Key for the data. If key already exists, data overwritten and consumers list will be updated.

  • data: required (file)

    Data to store. Max size is 1MB. Base64 encoded string.

  • consumers: (array of string)

    List of consumer IDs. If empty or not passed, current IYO_UID will be added.

Examples:

Simple put to private reservation:

{
  "IYO_UID": "6a6f686e646f656a756e696f7274776f",
  "reservationId": "D9kdMquM0ElYvnQ3",
  "dataSecret": "MqydbtJK/kdlJJ7RjW+5aS25ZlVKWvcWoLY2QXbEr6o=",
  "key": "EFBS-537",
  "data": "LongData...GoesHere"
}

Put to public reservation marked for multiple consumers:

{
  "IYO_UID": "6a6f686e646f656a756e696f7274776f",
  "reservationId": "MyPublicRes1",
  "key": "PUB-1004",
  "data": "LongData...GoesHere",
  "consumers": [
    "69845524-08c5-4870-a52e-b2878c67d779",
    "c80ed846-2bb0-46a9-8552-556e1d902431",
    "89d66e5e-9e3a-43e3-bcc0-9bcff6416493"
  ]
}

HTTP status code 200

Operation successfull

HTTP status code 403

reservationId is invalid or dataSecret is incorrect.

HTTP status code 413

Not enough space in reservation.

/nos/puts post

post

Put multiple objects to storage.

post /nos/puts

Put multiple objects to storage.

  • Request
  • Response

Body

Media type: application/json

Type: object

Properties
  • IYO_UID: required (string)

    Identity of requeter.

  • reservationId: required (string)

    Unique ID of the reservation.

  • dataSecret: (string)

    Secret with W access level to the specific reservation. Pass empty string or ignore if reservation is public.

  • payload: required (array of Payload)

    Array of Payload (key+data) to store in NOS.

    Items: Payload

    • key: required (string)

      Key of the Payload

    • data: required (file)

      Data to store. Max size is 1MB. Base64 encoded string.

  • consumers: (array of string)

    List of consumer IDs. If empty or not passed, current IYO_UID will be added. Same consumers set for all payloads passed.

Example:

Simple put to private reservation:

{
  "IYO_UID": "6a6f686e646f656a756e696f7274776f",
  "reservationId": "D9kdMquM0ElYvnQ3",
  "dataSecret": "MqydbtJK/kdlJJ7RjW+5aS25ZlVKWvcWoLY2QXbEr6o=",
  "payload": [
    {
      "key": "EFBS-537",
      "data": "LongData1...GoesHere"
    },
    {
      "key": "EFBS-538",
      "data": "LongData2...GoesHere"
    },
    {
      "key": "EFBS-539",
      "data": "LongData3...GoesHere"
    }
  ]
}

HTTP status code 200

Operation successfull

HTTP status code 403

reservationId is invalid or dataSecret is incorrect.

HTTP status code 413

Not enough space in reservation.

/nos/get post

post

Get object from NOS

post /nos/get

Get object from NOS

  • Request
  • Response

Body

Media type: application/json

Type: object

Properties
  • IYO_UID: required (string)

    Identity of requeter.

  • reservationId: required (string)

    Unique ID of the reservation.

  • dataSecret: (string)

    Secret with R access level to the specific reservation. Pass empty string or ignore if reservation is public.

  • key: required (string)

    Data key.

  • verify: (boolean)

    Verify data. If passed true, method uses crc to compare with existing data and returns true/false instead of actual data.

  • crc: (string)

    CRC to compare compare with existing data.

Examples:

Regular get for public data:

{
  "IYO_UID": "6a6f686e646f656a756e696f7274776f",
  "reservationId": "D9kdMquM0ElYvnQ3",
  "key": "EFBS-537"
}

Regular get for private data:

{
  "IYO_UID": "6a6f686e646f656a756e696f7274776f",
  "reservationId": "D9kdMquM0ElYvnQ3",
  "dataSecret": "MqydbtJK/kdlJJ7RjW+5aS25ZlVKWvcWoLY2QXbEr6o=",
  "key": "EFBS-537"
}

Private data verification request:

{
  "IYO_UID": "6a6f686e646f656a756e696f7274776f",
  "reservationId": "D9kdMquM0ElYvnQ3",
  "dataSecret": "MqydbtJK/kdlJJ7RjW+5aS25ZlVKWvcWoLY2QXbEr6o=",
  "key": "EFBS-537",
  "verify": true,
  "crc": "03938ba0"
}

HTTP status code 200

Returns data or crc verification

Body

Media type: application/json

Type: string

Examples:

Regular get request:

LongData1...GoesHere
[object Object]

CRC check request:

true

HTTP status code 403

reservationId is invalid or dataSecret is incorrect.

Properties

    "EFBS-537",
    "EFBS-538",
    "EFBS-539",
    "EFBS-540"
  ]
}

{
  "IYO_UID": "6a6f686e646f656a756e696f7274776f",
  "reservationId": "D9kdMquM0ElYvnQ3",
  "dataSecret": "MqydbtJK/kdlJJ7RjW+5aS25ZlVKWvcWoLY2QXbEr6o=",
  "keys": [
    "EFBS-537",
    "EFBS-538",
    "EFBS-539",
    "EFBS-540"
  ]
}

Private data verification request:

{
  "IYO_UID": "6a6f686e646f656a756e696f7274776f",
  "reservationId": "D9kdMquM0ElYvnQ3",
  "dataSecret": "MqydbtJK/kdlJJ7RjW+5aS25ZlVKWvcWoLY2QXbEr6o=",
  "keys": [
    "EFBS-537",
    "EFBS-538",
    "EFBS-539",
    "EFBS-540"
  ],
  "verify": true,
  "crcArray": [
    "03938ba0",
    "0c280ac4",
    "5b22776b",
    "c0764cbe"
  ]
}

Returns data or crc verification

Type: array of string

Examples:

Regular get request:

[
  "LongData1...GoesHere",
  "LongData2...GoesHere",
  "LongData3...GoesHere",
  "LongData4...GoesHere"
]

Some of the key not exists response:

[
  "",
  "LongData2...GoesHere",
  "",
  "LongData4...GoesHere"
]

CRC check request:

[
  "true",
  "true",
  "true",
  "true"

reservationId is invalid or dataSecret is incorrect.
