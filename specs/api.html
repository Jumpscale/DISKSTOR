post /reservations

Make a reservation for specific amount of storage in the NOS.

  • Request
  • Response

Headers

  • RequesterUID: required (string)

    Identity of sender.

    Example:

    6a6f686e646f656a756e696f7274776f

Body

Media type: application/json

Type: object

Properties
  • reservationId: (string)

    Unique id (upto 16 chars) which identifies the reservation. If not passed will be autogenerated by server.

  • reservationAdminSecret: (string)

    Secret to control the reservation. If not set - reservation is publicly available.

  • size: required (integer)

    Size of reservation in megabytes.

  • expirationEpoch: (integer)

    All data removed after expiration date. Epoh format.

Examples:

Regular reservation, reservationId generated by server and returned:

{
  "reservationAdminSecret": "MqydbtJK/kdlJJ7RjW+5aS25ZlVKWvcWoLY2QXbEr6o=",
  "size": 100
}

Private reservation with expiration:

{
  "reservationId": "D9kdMquM0ElYvnQ3",
  "reservationAdminSecret": "MqydbtJK/kdlJJ7RjW+5aS25ZlVKWvcWoLY2QXbEr6o=",
  "size": 50,
  "expirationEpoch": 1521532065
}

Public reservation:

{
  "reservationId": "MyPublicRes1",
  "size": 300
}

HTTP status code 200

Reservation successful

Body

Media type: application/json

Type: object

Properties
  • reservationId: required (string)

    reservationId passed in parameters or generated by the server if absent in parameters

Example:

{
  "reservationId": "D9kdMquM0ElYvnQ3"
}

HTTP status code 409

ReservationId is not unique.

HTTP status code 413

Not enough space available.

get /reservations/{reservationId}

Get infromation about reservation. Accessible with A (admin) rights for reservation.

  • Request
  • Response

URI Parameters

  • reservationId: required (string)

Headers

  • RequesterUID: required (string)

    Identity of sender.

    Example:

    6a6f686e646f656a756e696f7274776f
  • DataSecret: required (string)

    Data secret with proper access to the reservation.

    Example:

    MqydbtJK/kdlJJ7RjW+5aS25ZlVKWvcWoLY2QXbEr6o=

HTTP status code 200

Infromation fetched successfuly.

Body

Media type: application/json

Type: object

Properties
  • spaceReserved: required (integer)

    Space reserved for this reservation, in MB.

  • spaceUsed: required (integer)

    Space used in this reservation, in MB.

  • expirationEpoch: (integer)

    Expiration date in Epoh format

  • acls: required (array of AclEntry)

    List of ACL entries for the specific reservation.

    Items: AclEntry

    • dataSecret: required (string)

      Secret for which ACL is set.

    • acl: required (string)

      ACL settings. Can be a combination of letters R - read, W - write, D - deleate, A - admin.

  • nRequestsPerH: required (number)

    Number of requests per Hour for the specific reservation.

  • nObjestsStored: required (integer)

    Number of objects stored in specific reservation.

Example:

Reservation with expirationEpoch:

{
  "spaceReserved": 512,
  "spaceUsed": 34,
  "expirationEpoch": 1521532065,
  "acls": [
    {
      "dataSecret": "MqydbtJK/kdlJJ7RjW+5aS25ZlVKWvcWoLY2QXbEr6o=",
      "acl": "RWDA"
    },
    {
      "dataSecret": "MyPublicRes1",
      "acl": "RW"
    }
  ],
  "nRequestsPerH": 86,
  "nObjestsStored": 572
}

HTTP status code 403

DataSecret is invalid.

post /reservations/{reservationId}/objects

Put one or multiple objects into storage.

  • Request
  • Response

URI Parameters

  • reservationId: required (string)

Headers

  • RequesterUID: required (string)

    Identity of sender.

    Example:

    6a6f686e646f656a756e696f7274776f
  • DataSecret: required (string)

    Data secret with proper access to the reservation.

    Example:

    MqydbtJK/kdlJJ7RjW+5aS25ZlVKWvcWoLY2QXbEr6o=

Body

Media type: multipart/form-data

Type: object

Properties
  • keys: required (array of string)

    Keys for the data. If keys already exists, data overwritten and consumers list will be updated.

  • data: required (array of file)

    Data to store. Max size is 1MB. Base64 encoded string.

  • consumers: (array of string)

    List of consumer IDs (requesterUID). If empty or not passed, current requesterUID will be added.

Example:

Simple put to private reservation:

{
  "keys": [
    "EFBS-537",
    "EFBS-538",
    "EFBS-539"
  ],
  "data": [
    "LongData1...GoesHere",
    "LongData2...GoesHere",
    "LongData3...GoesHere"
  ],
  "consumers": [
    "69845524-08c5-4870-a52e-b2878c67d779",
    "c80ed846-2bb0-46a9-8552-556e1d902431",
    "89d66e5e-9e3a-43e3-bcc0-9bcff6416493"
  ]
}

HTTP status code 200

Operation successful

HTTP status code 403

DataSecret is invalid.

HTTP status code 413

Not enough space in reservation.
